In networking today, decentralization plays a more important role every day. While the trend in networking can be seen shifting towards edge-computing to reduce strain from major network hubs, challenges to maintain connections and provide guarantees towards their stability can arise. Due to this we propose a micro-segmentation approach of network resources across multiple domains that can provide the user with a slice of guaranteed resources while maintaining the privacy of the user.

Topology

Below you can see an example topology with all relevant components. Host 1 (h1) wants to communicate with Host 4 (h4).
This is just an example topology of many possible variations that can be created.

The Host wishes to maintain a connection to a host on another domain with guaranteed quality of service (QoS) parameters via a secure and private channel. It will only trust his own and the network of the recipient. To be able to connect to the other host, the host will contact the ESMF of his domain to request a slice through the network to the other host.
The Edge Slice Management Function (ESMF) is used by hosts to register a communication intention. Once an intention with requirements is received, the ESMF performs a two-phase commit protocol with other ESMFs and CTMFs to create the requested communication channel if it is able to.
The Core Tunnel Management Function (CTMF) is used by ESMFs to allocate resources in a black network (a network that is not managed by us). The CTMF will not receive the details of the slice but will rather see the sum of all the needed capacity. Due to this we can achieve more privacy from our transit networks. The CTMF is a subset of the ESMF.
The Domain Slice Management Function (DSMF) is instructed by the ESMF to reserve and create new slices in an edge network. It will instruct the domain switches to realise the slice within the domain of the function. Additionally, it also provides the ability to create tunnels like the DTMF. Upon tunnel creation it will also (optionally) set up a tunnel entry on a vpn gateway of the edge to safely encrypt the transferred data before it enters the first black network.
The Domain Tunnel Management Function (DTMF) is instructed by the CTMF to reserve and create new tunnel capacity in a black network. It will instruct the domain switches to realise the tunnel capacity within the domain of the function. The DTMF is a subset of the DSMF.
The Switch provides the ability to create and remove flows (to the controller via OpenFlow), as well as create and remove quality of service (QoS) queues to the DSMFs and DTMFs.
The Controller provides the ability to create and remove flows on the switches connected to it. It exposes this functionality to the DSMFs and DTMFs.
The VPN Gateway provides the ability to create and remove tunnel entries. These tunnel entries are used to encrypt data before it enters the first black network. The black networks then only know that some data is transferred using a specific tunnel, but not what the data contains or which slice it belongs to.

Routing

Our routing rules within a setup can be viewed below. These rules are conceptualized with other competing traffic in mind. Please note that the vpn gateways do not forward any traffic apart from traffic that will enter a tunnel. This is why we display connections from s3 to s4 and s6 to s7 here. All components starting with src_, bn_ (black network) or dst_ can be merged if necessary.
